SLE 2014 SI paper:
Practical domain-specific debuggers using the Moldable Debugger framework

Andrei Chiș, Marcus Denker, Tudor Gîrba, Oscar Nierstrasz

  • DOI: 10.1016/j.cl.2015.08.005
  • Extended version of The Moldable Debugger: A Framework for Developing Domain-Specific Debuggers (SLE 2014)
  • T4A: Workbenches
    • The central contribution is a framework (tooling/workbench support) for constructing domain-specific debuggers via reusable operations and views.
  • T4D: Interpretation
    • The work is about inspecting and interacting with running program executions (debugging runtime behavior).
  • T5H: Usability
    • A primary motivation is improving developer workflow and reducing the abstraction gap/cognitive load through tailored debugging views.
  • T5F: Performance
    • The paper discusses implementation strategies and their performance impact (notably overhead being opt-in).
