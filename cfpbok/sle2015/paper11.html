SLE 2015 paper:
Supporting efficient and advanced omniscient debugging for xDSMLs

Erwan Bousse, Jonathan Corley, Benoît Combemale, Jeffrey G. Gray, Benoît Baudry

  • DOI: 10.1145/2814251.2814262
  • T4A: Workbenches
    • The paper’s primary contribution is debugging tooling (omniscient debugging) for executable DSMLs.
  • T4D: Interpretation
    • Omniscient debugging relies on runtime execution tracing and state navigation.
  • T5H: Usability
    • The contribution is largely about improving developer interaction with executions (debugging usability).
  • T3C: DSLs
    • The target artifacts are executable domain-specific modeling languages (xDSMLs).
