PV4 tools in ProVerB

PV4 tools allow the end user to write their own specifications and “compile” them to fully formal mathematical representations that can be combined with mathematical representations of programs themselves or their desired properties, and verified together. Such a tool can already help someone make claims and guarantees about correctness, complexity or predicted performance of a novel, previously non-existing, garbage collection algorithm. Some of PV4 tools focus on synthesising a software artefact that satisfies the desired conditions, instead of checking an existing one for conformance.
  • ABC (versatile verification and synthesis arsenal)
  • AIGEN (synthesises a transition system from the set of requirements)
  • Alt-Ergo (produces a satisfiability result for a formula)
  • AMUSIC (counts minimal unsatisfiable subsets of a given formula in CNF)
  • AMYTISS (synthesises a spec-conforming controller from a Markov decision process)
  • Attestor (checks if a property holds for a program, generates additional properties, provides examples and analysis logs)
  • Avy (takes a model and a property and checks satisfiability within bounds)
  • Barcelogic (produces a satisfiability result for a formula)
  • Boolector (produces a satisfiability result for a formula)
  • CoqQFBV (produces a satisfiability formula for a query)
  • CryptoMiniSat (produces a satisfiability result for a formula in CNF)
  • CVC4 (produces a satisfiability result for a formula)
  • cvc5 (produces a satisfiability result for a formula)
  • DaPPer (synthesises probabilistic programs from relational datasets)
  • dReal (produces a satisfiability result for a formula)
  • EAHyper (checks satisfiability, implication and equivalence of hyperproperties)
  • GDVB (given a problem and constraints, generates benchmarks)
  • GenMC (given a C program with assertions and a memory model spec, reports safety errors)
  • Glucose (produces a satisfiability result for a formula in CNF)
  • Kissat (produces a satisfiability result for a formula)
  • Lingeling (produces a satisfiability result for a formula)
  • MathSAT (produces a satisfiability result for a formula)
  • MCHyper (proves given properties on a given circuit)
  • MightyL (compiles a temporal logic formula to a timed automaton)
  • MiniSat (produces a satisfiability result for a formula)
  • MiniSat+ (produces a satisfiability result for a formula)
  • MonoSAT (produces a satisfiability result for a formula)
  • ParaFROST (produces a satisfiability result for a formula)
  • PicoSAT (produces a satisfiability result for a formula)
  • plingeling (produces a satisfiability result for a formula)
  • Q3B (produces a satisfiability result for a formula)
  • Reluplex (produces a satisfiability result for a formula)
  • SMTInterpol (produces a satisfiability result for a formula)
  • SPIN (analyses models of multi-threaded applications with different message passing paradigms or shared memory communication)
  • Storm (computes how a given property is related to a given specification)
  • Yices (produces a satisfiability result for a formula)
  • Z3 (produces a satisfiability result for a formula)
  • zChaff (produces a satisfiability result for a formula in CNF)
