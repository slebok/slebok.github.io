PV3 tools in ProVerB

PV3 tools operate within a particular paradigm and derive their actions, conclusions and even properties that they need to check, from a built-in formal specification. For example, such a tool can guarantee freedom of deadlocks in a multi-threaded application, or data consistency in a database management system. Either the specification or the property set is fixed for tools of this level.
  • 2LS (can generate properties within certain conditions)
  • Boogie (infers invariants and generates verification conditions to pass to Z3)
  • BoSy (proves realisability of a given property)
  • BoSyHyper (proves realisability of a given hyperproperty)
  • cake_lpr (takes a proof and checks whether it satisfies linear propagation redundancy expectations)
  • DLC (compiles a formal specification of a concurrent system to a program)
  • NNV (checks whether the given network is safe)
  • PAF (computers sound but tight probabilistic bounds to roundoff errors in arithmetic expressions)
  • Vampire (proves a theorem in first order logic)
